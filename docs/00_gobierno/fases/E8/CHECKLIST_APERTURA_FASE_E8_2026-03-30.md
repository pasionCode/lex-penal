Esta apertura autoriza exclusivamente:

1. Definir una suite de regresión mínima post-cierre MVP.
2. Revisar variables de entorno, secretos y configuración sensible.
3. Levantar estrategia de despliegue objetivo.
4. Preparar checklist preproducción.
5. Identificar riesgos operativos de salida a entorno.

## 5. Orden de trabajo previsto
1. Regresión mínima
2. Hardening de configuración
3. Estrategia de despliegue
4. Checklist preproducción
5. Decisión de salida

## 6. Fuera de alcance
- Nuevas funcionalidades de negocio
- Rediseños de arquitectura no justificados por validación operativa
- Cambios de producto no ligados a despliegue o estabilidad
- Expansión funcional del MVP

## 7. Criterios de entrada
- MVP cerrado formalmente
- Fase E7 cerrada
- Repositorio limpio
- Contrato API y gobierno consolidados

## 8. Criterios de salida de fase
La fase E8 solo podrá cerrarse cuando exista, como mínimo:

1. evidencia de regresión mínima ejecutada;
2. revisión de configuración y secretos;
3. estrategia de despliegue documentada;
4. checklist preproducción emitido;
5. decisión formal sobre siguiente paso operativo.

## 9. Riesgos controlados
- Salida a despliegue sin validación mínima
- Configuración insegura o incompleta
- Falta de criterio de rollback o preparación operativa
- Reapertura desordenada de cambios funcionales
